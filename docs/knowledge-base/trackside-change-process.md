title: Trackside Change Process
tags: [process, changes, workflow]
---

At the track, the fastest path to a usable setup is usually the most disciplined one. Change one thing at a time, write it down, and connect the rider feedback to a specific phase of the corner or braking zone. This does more for future speed than making a larger number of random adjustments.

A strong trackside workflow starts by identifying the symptom precisely. Is it entry stability, mid-corner push, exit grip, braking support, or curb compliance? The next step is to check whether a simple baseline issue could explain it: tire pressure, sag, ride height, tire condition, or a recent obvious setup change.

Only after that should you make a single controlled adjustment. Keep the change small enough that the rider can actually feel the difference without moving the setup to a new problem entirely. Then compare the result against the prior session rather than relying on memory alone.

If the rider wants to change multiple things at once, resist it unless the current setup is clearly unusable. Multiple changes destroy the learning value of the session log. When the bike improves, you do not know which lever helped. When it gets worse, you do not know which one to undo.

The session log is the memory system. It turns a track day from a string of impressions into a usable tuning history.
